## 🔧 How to Deploy

### **Option 1: Render (RECOMMENDED - Easiest)**
1. Push to GitHub
2. Connect repo to Render.com
3. Create new Web Service
4. Render auto-detects `render.yaml`
5. Automatic PostgreSQL, migrations, SSL ✓

### **Option 2: Vercel**
1. Set environment variables in Vercel dashboard
2. `vercel --prod`
3. (Need external PostgreSQL: Render, Supabase, AWS RDS)

### **Option 3: Heroku**
1. `heroku create app-name`
2. `heroku addons:create heroku-postgresql:mini`
3. Set config vars
4. `git push heroku main`

### **Option 4: Docker (VPS)**
1. Clone repo to VPS
2. `docker-compose up -d`
3. `docker-compose exec web python manage.py migrate`

## 📋 Critical Setup Steps Before Deployment

1. **Generate new SECRET_KEY:**
   ```bash
   python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
   ```

2. **Create .env file:**
   ```bash
   cp .env.example .env
   # Edit with production values
   ```

3. **Set your domain in ALLOWED_HOSTS**

4. **Test with PostgreSQL locally:**
   ```bash
   docker-compose up -d
   docker-compose exec web python manage.py migrate
   ```

5. **Run deployment check:**
   ```bash
   DEBUG=False DJANGO_SECRET_KEY="test" ALLOWED_HOSTS="localhost" \
   python manage.py check --deploy
   ```
